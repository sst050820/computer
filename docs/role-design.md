# 农禾坊 — 五角色功能设计

## 角色总览

| 角色 | ABE 对应 | 核心职责 |
|------|---------|----------|
| 消费者 | DataOwner | 浏览商品、发布定制需求（设定 ABE 访问策略） |
| 商家 | DataUser | 商品管理、查看匹配需求（持有属性密钥） |
| 审核方 | AttributeAuthority | 审核颁发资质（属性密钥分发）、收回资质（属性撤销） |
| 管理员 | 超级用户 | 全局管理、用户管理、系统密钥轮换（SysUpdate） |
| 监管方 | 监管审计 | 产品追溯、合规检查、应急解密 |

### ABE 解密流程

1. 点击需求卡片 → 弹出详情
2. 点击「ABE 解密验证」→ 调 `POST /api/custom-orders/:id/decrypt`
3. 传入商家ID → 后端获取其有效资质 → 调 Java ABE 解密
4. 成功：绿框显示原文内容 → 可接单报价
5. 失败：红框显示"资质不满足条件" → 引导申请资质

### ABE 属性撤销

- 点击「收回」→ 确认弹窗(含ABE撤销警告)
- 调 RevokeQualification → DB: status='revoked'
- 调 RevokeAttribute → Java ABE `/api/revoke`
- 返回消息含ABE撤销状态

### ABE 系统密钥轮换

- 点击「更新认证规则」→ 调 `POST /api/admin/sys-update`
- UpdateSystemKeys() → Java ABE `/api/rekey`
- ExpireAllQualifications() → 所有active资质→expired
- 前端显示ABE密钥更新状态(成功/降级)
